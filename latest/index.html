Discover API Features

Learn about different features Minecraft Script API provides, and explore the reference documentation to bring your idea to life.

World Manipulation

Exposing essential properties and functionalities and manipulate elements like blocks, items, entities and other content in a Minecraft world. Examples include:

  • Retrieves and change players' Gamemode with Gamemode APIs
  • Read and write saved structures using Structure APIs
  • Clearing the inventory with Inventory APIs

Script-Command Bridge

Minecraft's Script API provides a bridge between script and command logic. The /scriptevent command triggers a script event with an ID and message, which can be responded by registering the scriptEventReceive event handler with scripts.

Storing Data in a Minecraft World

The Minecraft module also provides mechanisms for script plugins to store data. The data are stored in a Minecraft world in key/value pairs.

The two mechanisms within Minecraft world storage (accessible through Script API) are as follows:

  • Dynamic Properties: Maintains a separate storage for entities, items and world, that is accessible with only script API per behavior pack.
  • Scoreboards: Stores the score of entities in objectives. An objective tracks a single statistic for entities.
  • Entity Properties: Save data or store values on entities that can be optionally read by Resource Packs.

Basic UI forms

Build simple forms or dialogs with @minecraft/server-ui module, and send them to players. This allows script plugins to collect form-like data in Minecraft.

  • Action Forms: A form with a list of buttons with captions and images for presenting a set of options to a player.

  • Message Forms: A form with two-button that are functional for Yes/No or OK/Cancel questions.

  • Modal Forms: A more flexible "questionnaire-style" list of controls in a form that can be used to take input.

GameTest Framework

One of the sample GameTests from Microsoft's sample GameTest repository on GitHub

A toolset available to creators to build and test any new content using the @minecraft/server-gametest script module, and the /gametest command in Minecraft.

Editor Extensions

Bedrock Editor with Editor Samples Add-On in Minecraft Preview v1.21.30 (August 2024)

An Editor extension is a set of resource packs and behavior packs, like .mcaddon files but for Editor, uses the Minecraft Editor API to perform most actions in the Bedrock Editor. They enable users to use complex tools to build, edit, and test worlds in their preferences.

@minecraft/server

The script module which is essential for managing and interacting with a Minecraft world, including:

  • Exposing essential properties and functionalities for entities, blocks, dimensions, and more.
  • Listening for World or System events.
  • Retrieves system information of a client and a server with Device Info APIs
  • Listen for input events and information using Input APIs.

@minecraft/server-ui

Links

  • Manifest Details (Latest stable)

  • Installation (Latest stable)

A module for creating simple pop-up forms that allow users to provide you with formatted inputs through submissions. The three main form types are: ActionFormData, MessageFormData and ModalFormData.

@minecraft/server-gametest

The module behind GameTest Framework, it's used to manage all GameTests within an Add-On.

Each GameTest is broken into the following:

  • Register a test: create an environment to conduct a test.
  • Setup a test: Setup the environment to run the test.
  • Validate a test: Evaluate whether a test passed or failed.

@minecraft/server-editor

The Minecraft Editor module for Editor extensions to build complex tools to build, edit, and test worlds. It is a built-in JavaScript library.

Currently, this API is only accessible in Bedrock Preview on Windows.

Here are some examples of what you can achieve with the Extension API:

  • Add built-in Ore UI components & views in the editor UI.
  • Create custom tools in the Editor panel.
  • Register custom actions and hotkeys to use a tool.
  • Export and playtest an Editor project as a playable world.

@minecraft/server-net

The module which allows for server-based HTTP request handling, and server packets handling within a Bedrock Dedicated Server.

@minecraft/server-admin

Manages the administration in a Bedrock Dedicated Server. Its current capabilities include:

  • Load configuration of variables and secrets in the Bedrock Dedicated Server folder.
  • Transfer a player to another Bedrock server.

@minecraft/debug-utilities

A Script API module that is used for debugging purposes. Currently features:

  • Configure performance watchdog timing warnings for a plugin.
  • Provide access to performance-related information for script runtimes.
